## 一、项目定位

MOYU 是一个**轻量、框架无关的 AI Agent 记忆工具包**。

- 目标用户：个人开发者、小团队
- 理念：做对一件事做到极致，不堆砌功能
- 边界：不破坏其他模块，核心代码保持框架无关
- 选型：不走平台绑定路线（不学大公司做生态锁定），即插即走

## 二、发布流程

### GitHub Release

1. 修改 `updater.py` 中的 `VERSION` 和 `_CHECKSUMS`
2. 先算好本地的 `git archive` zip SHA256，再 commit（避免先 commit 再改 checksum 需要 amend）
   ```bash
   git archive --format=zip --prefix=moyu_toolkit/ -o /tmp/moyu-toolkit-X.X.X.zip vX.X.X:moyu_toolkit/
   # 算 SHA256
   ```
3. commit + tag + push
   ```bash
   git add -A
   git commit -m "vX.X.X — 变更说明（英文）"
   git tag -a vX.X.X -m "vX.X.X"
   git push origin main --tags
   ```
4. GitHub Release 页面创建 Release：
   - 标题：`vX.X.X — 一句话描述`
   - 描述：英文，只列实际变更，不画饼
   - 链接：`https://github.com/awchzh/moyu-memory/releases/new?tag=vX.X.X`

### SkillHub 上传

1. 从 tag 打包，只打 `moyu_toolkit/` 目录（不要整个仓库）
   ```bash
   git archive --format=zip --prefix=moyu_toolkit/ -o /tmp/moyu-toolkit-X.X.X.zip vX.X.X:moyu_toolkit/
   ```
2. 确认 zip 包含 `moyu_toolkit/SKILL.md`
3. 变更说明：中文，和 GitHub 对应的内容
4. 上传到 SkillHub，版本号与 GitHub 一致

### 注意事项

- GitHub 描述用英文，SkillHub 用中文
- 两个平台的版本号保持一致
- updater.py 的 `_CHECKSUMS` 需要在 Release 前就写入正确的 SHA256
- ZIP 只含 `moyu_toolkit/`，不含 LICENSE/README 等根目录文件（SkillHub 要求）

## 四、代码架构

```
moyu_toolkit/
├── agent_memory.py      # TEMPR 多策略检索（语义 + BM25 + 时间）
├── active_context.py    # 工作记忆管理
├── knowledge_graph.py   # 轻量知识图谱
├── learner.py           # 从纠正中学习 + 用户画像
├── security.py          # 记忆自我保护（密码验证）
├── defense_toolkit/
│   └── integrity_checker.py  # 完整性校验 + 自动恢复 + 法医分析
├── context_manager.py   # 上下文自适应压缩
├── forgetting_curve.py  # 遗忘曲线
├── memory_merge.py      # 记忆合并
├── session_bridge.py    # 跨会话桥接（内部工具，非发布能力）
├── self_reflection.py   # 自我反思（矛盾检测+关联发现，V2.0.3）
├── updater.py           # 自动更新（含 SHA256 校验）
├── moyu.py              # 统一 CLI 入口
├── moyu_demo.py         # Demo 模式
├── moyu_wake.py         # 唤醒例程
├── config.yaml          # 配置
├── SKILL.md             # SkillHub 描述文档
├── requirements.txt     # 依赖
└── 墨羽项目.md          # 项目知识库（本文件）
```
